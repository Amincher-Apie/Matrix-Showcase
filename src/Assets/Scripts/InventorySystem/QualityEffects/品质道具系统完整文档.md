## 系统概述

品质道具效果系统（TCA系统）是一个完全数据驱动的道具效果框架，允许策划在 Unity Inspector 中配置复杂的道具效果，无需修改代码。

### 核心优势

- **数据驱动**：所有效果通过 ScriptableObject 配置，策划可直接编辑
- **可组合**：任意 Trigger + Condition + Action 自由组合
- **可扩展**：新增效果类型只需实现一次执行器，所有道具可复用
- **可堆叠**：自动支持层数叠加，参数随层数缩放
- **策划友好**：下拉列表选择，中英文对照显示，避免拼写错误
- **服务器权威**：所有逻辑在服务器执行，防止作弊

### 系统流程

```
战斗事件发生
    ↓
Trigger（触发器）→ 何时检查效果
    ↓
Condition（条件）→ 是否满足条件（AND 逻辑，全部通过才执行）
    ↓
Action（动作）→ 执行什么效果（顺序执行）
```

## 核心概念

### TCA 模式

- **Trigger（触发器）**：定义何时检查效果（如：造成伤害时、击杀敌人时）
- **Condition（条件）**：定义是否满足触发条件（如：20%概率、血量低于30%）
- **Action（动作）**：定义执行什么效果（如：造成伤害、施加Buff、治疗）

### 效果合并策略

通过**标签（Tags）**控制相同效果ID的道具如何合并：

- **无标签或 `NoMerge`**：每个道具单独执行，数值累加（如攻速加成）
- **`MergeByQuality`**：相同效果ID按品质分组合并，层数=该品质的道具数量（如概率效果）
- **`perQuality`**：相同效果ID合并所有品质，层数=所有道具总数（如持续时间效果）

### 参数缩放

通过 `perStack` 参数控制哪些属性随层数增加：

- **配置了 `perStack` 的参数**：会随层数增加（如 `chance.perStack`, `duration.perStack`）
- **未配置 `perStack` 的参数**：保持基础值不变

## 系统架构

### 核心文件结构

```
QualityEffects/
├── QualityEffectDefinitions.cs      # 效果定义数据结构
├── QualityEffectConstants.cs        # 常量定义（触发器、条件、动作ID）
├── QualityEffectRegistry.cs         # 执行器注册表
├── Executors/
│   ├── ConditionExecutors.cs        # 条件执行器实现
│   └── ActionExecutors.cs          # 动作执行器实现
└── PlayerQualityEffectModule.cs    # 效果管理模块（在Framework中）
```

### 数据流

1. **配置阶段**：策划在 Unity Inspector 中配置 `QualityItemSO` 的效果列表
2. **初始化阶段**：`PlayerQualityEffectModule.LocalInit()` 注册所有执行器
3. **装备阶段**：玩家装备道具时，`RebuildFromInventory()` 重建效果列表
4. **触发阶段**：战斗系统调用 `RaiseOnHitDealt()` 等方法触发效果
5. **执行阶段**：系统检查触发器→条件→执行动作

## 配置指南

### 在 Unity Inspector 中配置效果

1. 创建或打开 `QualityItemSO` 资源
2. 在 Inspector 中找到 **"TCA 效果列表"** 折叠栏
3. 点击 **"+"** 添加新效果
4. 配置效果的各个字段

### 效果定义字段说明

#### 基础字段

| 字段 | 说明 | 示例 |
|------|------|------|
| **效果ID** | 唯一标识符，用于冷却和检测 | `kill_heal_chance` |
| **标签** | 逗号分隔的标签，控制合并策略 | `MergeByQuality`、`MergeAll`、`NoMerge` |
| **最大层数** | 该效果最多可叠加多少层 | `99`（默认） |
| **冷却时间(秒)** | 效果触发后的冷却时间 | `0`（无冷却） |

#### 触发器列表

- 点击 **"+"** 添加触发器
- 从下拉列表选择触发器（显示中英文对照）
- 支持多个触发器（OR 逻辑，任一匹配即可）
- 定时触发器需要手动输入：`OnTick:1.0`（1秒触发一次）

#### 条件列表

- 点击 **"+"** 添加条件
- 从下拉列表选择条件ID（显示中英文对照）
- 配置该条件所需的参数
- 所有条件必须全部通过（AND 逻辑）

#### 动作列表

- 点击 **"+"** 添加动作
- 从下拉列表选择动作ID（显示中英文对照）
- 配置该动作所需的参数
- 按顺序执行所有动作

### 参数配置

#### 参数配置界面

每个参数配置项包含：

1. **参数名**：下拉列表选择或手动输入（显示中英文对照）
2. **参数类型**：选择 `Float`（浮点数）或 `String`（字符串）
3. **参数值**：根据类型输入对应值
   - Float类型 → 输入浮点数（如：`0.2`、`5.0`）
   - String类型 → 输入字符串（如：`"Slow"`、`"Rage"`）

#### 参数缩放语法

所有浮点参数支持以下缩放方式：

```
最终值 = (base + perStack * stacks + perQuality * quality) * mult
```

**参数键名格式**：
- `参数名` 或 `参数名.base`：基础值
- `参数名.perStack`：每层增加值
- `参数名.perQuality`：每品质增加值
- `参数名.mult`：最终倍数

**示例**：
- `chance.base = 0.1`：基础 10% 几率
- `chance.perStack = 0.1`：每层 +10% 几率
- 1 层：10%，2 层：20%，3 层：30%

## 效果叠加策略

### 策略说明

相同效果ID的效果会根据标签策略自动合并，层数=道具数量。

#### 1. NoMerge（不合并）

**标签**：无标签或 `NoMerge`

**行为**：每个道具单独执行，数值累加

**适用场景**：线性叠加效果（如攻速加成、伤害加成）

**示例**：
- 2个绿色士兵注射器（每个+15%攻速）+ 1个蓝色士兵注射器（+20%攻速）
- 最终效果：50%攻速（15+15+20），每个效果分别执行

#### 2. MergeByQuality（按品质合并）

**标签**：`MergeByQuality`

**行为**：相同效果ID按品质分组合并，层数=该品质的道具数量

**适用场景**：概率类效果，希望相同品质合并，不同品质分别执行

**示例**：
- 2个绿色（20%概率）+ 1个蓝色（30%概率）
- 绿色组：层数2 → 40%概率回血（执行一次判断）
- 蓝色组：层数1 → 30%概率回血（执行一次判断）
- 总共执行两次判断，可能触发两次回血

#### 3. MergeAll（合并所有）

**标签**：`MergeAll`

**行为**：相同效果ID合并所有品质的道具，层数=所有道具总数

**适用场景**：持续时间类效果，希望所有道具合并计算层数

**示例**：
- 2个道具（减速2秒）→ 合并为层数2
- `duration.perStack = 2` → 最终持续时间 = 0 + 2×2 = 4秒

## 参数缩放系统

### 缩放公式

```
最终值 = (base + perStack * stacks + perQuality * quality) * mult
```

### 参数键名规则

| 键名格式 | 说明 | 示例 | 优先级 |
|---------|------|------|--------|
| `参数名` | 基础值（简化写法） | `chance = 0.2` | 低（如果同时存在 `.base` 则忽略） |
| `参数名.base` | 基础值（明确写法） | `chance.base = 0.2` | 高（优先使用） |
| `参数名.perStack` | 每层增加值 | `chance.perStack = 0.1` | - |
| `参数名.perQuality` | 每品质增加值 | `duration.perQuality = 1.0` | - |
| `参数名.mult` | 最终倍数 | `damage.mult = 1.5` | - |

#### `参数名` vs `参数名.base` 的区别

**功能上等价**：两者都作为基础值使用，在解析时：
1. 优先读取 `参数名.base`
2. 如果没有 `参数名.base`，则读取 `参数名`
3. 如果都没有，使用默认值

**使用建议**：
- **只配置基础值**：使用 `参数名`（简化写法）
  - 例如：`chance = 0.2`
- **需要配置缩放参数**：使用 `参数名.base`（明确写法，更清晰）
  - 例如：`chance.base = 0.2` + `chance.perStack = 0.1`
- **不要同时配置两者**：如果同时配置了 `chance` 和 `chance.base`，系统会优先使用 `chance.base`，`chance` 会被忽略

**示例对比**：

```csharp
// 方式1：简化写法（推荐用于简单场景）
chance = 0.2
// 解析结果：基础值 = 0.2

// 方式2：明确写法（推荐用于需要缩放的场景）
chance.base = 0.2
chance.perStack = 0.1
// 解析结果：基础值 = 0.2，每层 +0.1

// 方式3：同时配置（不推荐）
chance = 0.3
chance.base = 0.2
chance.perStack = 0.1
// 解析结果：基础值 = 0.2（优先使用 .base），chance = 0.3 被忽略
```

### 运行时上下文

参数解析时会使用以下上下文信息：

- `stacks`：效果层数（道具数量）
- `quality`：道具品质（0=白色, 1=绿色, 2=蓝色, 3=紫色, 4=红色）
- `baseDamage`：基础伤害值
- `targetMaxHp`：目标最大生命值
- `ownerLevel`：拥有者等级

### 配置示例

**示例1：概率随层数增加**
```
chance.base = 0.2        (基础20%)
chance.perStack = 0.1    (每层+10%)
层数1：20%
层数2：30%
层数3：40%
```

**示例2：持续时间随层数和品质增加**
```
duration.base = 2.0           (基础2秒)
duration.perStack = 1.0        (每层+1秒)
duration.perQuality = 0.5      (每品质+0.5秒)
绿色品质，层数2：2 + 1×2 + 0.5×1 = 4.5秒
蓝色品质，层数2：2 + 1×2 + 0.5×2 = 5.0秒
```

## 条件列表

所有条件在下拉列表中显示为：**"中文名称 (英文ID)"**

所有条件必须全部通过（AND 逻辑），任一失败则跳过该效果。

| 条件ID | 中文名称 | 参数 | 说明 |
|--------|---------|------|------|
| `RandomChance` | 随机几率 | `chance` | 随机概率判断 |
| `HPRatioLessThan` | 血量比例低于阈值 | `threshold`, `target` | 血量比例 < 阈值 |
| `HPRatioGreaterThan` | 血量比例高于阈值 | `threshold`, `target` | 血量比例 > 阈值 |
| `HasStatus` | 拥有特定状态 | `statusId`, `target` | 目标带有特定状态 |
| `NoStatus` | 不拥有特定状态 | `statusId`, `target` | 目标不带有特定状态 |

### 条件参数说明

#### RandomChance（随机几率）

**参数**：
- `chance` 或 `chance.base`：基础概率（0-1）
- `chance.perStack`：每层增加的概率
- `chance.perQuality`：每品质增加的概率
- `chance.mult`：概率倍数

**示例**：
```
chance.base = 0.2
chance.perStack = 0.1
层数1：20%概率
层数2：30%概率
```

#### HPRatioLessThan / HPRatioGreaterThan（血量比例）

**参数**：
- `threshold`：阈值（0-1）
- `target`：目标（"Self" 或 "Target"，默认 "Self"）

**示例**：
```
threshold = 0.3
target = "Self"
当自身血量比例 < 30% 时触发
```

#### HasStatus / NoStatus（状态检查）

**参数**：
- `statusId`：状态ID（字符串）
- `target`：目标（"Self" 或 "Target"，默认 "Self"）

**示例**：
```
statusId = "Burning"
target = "Target"
检查目标是否带有"燃烧"状态
```

#### RecentKills（最近击杀数）

**参数**：
- `count`：需要的击杀数
- `window`：时间窗口（秒）

**示例**：
```
count = 3
window = 1.0
1秒内击杀3个敌人时触发
```

### 动作参数说明

#### AddStat（增加属性）

**参数**：
- `stat`：属性名（字符串，如 "MaxHP"、"Attack"）
- `amount.base`：基础数值
- `amount.perStack`：每层增加数值

**示例**：
```
stat = "MaxHP"
amount.base = 25
amount.perStack = 25
层数1：+25最大生命
层数2：+50最大生命
```

#### AddWeaponStat（增加武器属性）

**参数**：
- `stat`：武器属性名（字符串，如 `RangeMax`、`FireDamage`、`SolidDamage`）
- `amount`：修改数值
- `modifyType`：武器修改类型（可选，如 `Range`、`SpecificElement`）
- `operator`：修改操作符（可选，默认 `Add`；`Percentage` 会映射为 `Percent`）

**示例**：
```
stat = "RangeMax"
amount.base = 5
modifyType = "Range"
operator = "Add"
```

#### RemoveWeaponStat（移除武器属性）

**参数**：
- `stat`：武器属性名，必须与 `AddWeaponStat` 使用的 `stat` 一致

移除时会使用 `effectId + stat + ownerId` 生成的 sourceId 精确撤销对应武器修改器。

#### ApplyDot（施加持续伤害）

**参数**：
- `damagePctOfBase`：伤害百分比（基于基础伤害）
- `duration`：持续时间（秒）

**示例**：
```
damagePctOfBase = 2.4
duration = 5
造成基础伤害240%的持续伤害，持续5秒
```

#### ApplyBuff（施加增益状态）

**参数**：
- `buffId`：Buff ID（字符串）
- `duration`：持续时间（秒）
- `moveSpeedPct`：移速百分比加成
- `attackSpeedPct`：攻速百分比加成
- `damagePct`：伤害百分比加成

**示例**：
```
buffId = "Rage"
duration.base = 6
duration.perStack = 4
moveSpeedPct = 0.5
attackSpeedPct = 1.0
施加"狂暴"状态，移速+50%，攻速+100%
```

#### ApplyDebuff（施加减益状态）

**参数**：
- `debuffId`：Debuff ID（字符串）
- `duration`：持续时间（秒）
- `slowPct`：减速百分比（0-1）
- `armorReduction`：护甲削减

**示例**：
```
debuffId = "Slow"
slowPct = 0.6
duration.base = 0
duration.perStack = 2
减速60%，持续时间随层数增加
```

#### Heal（瞬时治疗）

**参数**：
- `amount`：治疗量

**示例**：
```
amount = 50
立即回血50点
```

## 实战配置示例

### 示例1：击杀概率回血（按品质合并）

**需求**：击杀敌人有概率回血，多个相同品质道具的概率值合并，不同品质分别执行。

**配置步骤**：

1. **效果ID**：`kill_heal_chance`
2. **标签**：`MergeByQuality`
3. **触发器**：选择 `击杀敌人时 (OnKill)`
4. **条件**：
   - 条件ID：选择 `随机几率 (RandomChance)`
   - 参数配置：
     - 参数名：`概率-基础值 (chance.base)`，类型：Float，值：`0.2`
     - 参数名：`概率-每层增加 (chance.perStack)`，类型：Float，值：`0.2`
5. **动作**：
   - 动作ID：选择 `瞬时治疗 (Heal)`
   - 参数配置：
     - 参数名：`数值 (amount)`，类型：Float，值：`50`

**效果说明**：
- 绿色品质道具（20%概率）：
  - 1个：层数1 → 20%概率回血
  - 2个：层数2 → 40%概率回血（执行一次判断）
  - 3个：层数3 → 60%概率回血（执行一次判断）
- 蓝色品质道具（30%概率，假设配置了 `chance.base = 0.3`）：
  - 1个：层数1 → 30%概率回血
  - 2个：层数2 → 60%概率回血（执行一次判断）
- 混合装备：2个绿色（20%概率）+ 1个蓝色（30%概率）
  - 绿色组：层数2 → 40%概率回血（执行一次判断）
  - 蓝色组：层数1 → 30%概率回血（执行一次判断）
  - 总共执行两次判断，可能触发两次回血

## 注意事项

### 配置注意事项

1. **效果ID必须唯一**：相同效果ID的效果会根据标签策略合并
2. **定时触发器**：`OnTick:1.0` 这种格式需要手动输入（因为数值部分可变）
3. **自定义参数**：如果参数名不在下拉列表中，可以手动输入
4. **参数类型**：确保参数类型（Float/String）与值匹配

### 性能注意事项

1. **效果数量**：避免配置过多效果，影响性能
2. **冷却时间**：合理设置冷却时间，避免频繁触发
3. **条件检查**：条件检查是按顺序执行的，将最可能失败的条件放在前面

### 调试建议

1. **查看日志**：所有执行器都有 Debug.Log 输出，可以在 Console 中查看
2. **检查层数**：确认效果的层数是否正确计算
3. **检查参数**：确认参数是否正确解析

## 常见问题

### Q: 为什么我的效果没有触发？

A: 检查以下几点：
1. 触发器是否正确配置
2. 条件是否全部通过
3. 效果是否有冷却时间
4. 效果ID是否重复导致合并

### Q: 如何让效果不随层数增加？

A: 不配置 `perStack` 参数即可。例如：
- 配置 `chance.base = 0.2`，不配置 `chance.perStack`
- 无论层数多少，概率都是20%

### Q: 如何让不同品质的道具效果不同？

A: 使用 `perQuality` 参数。例如：
- `duration.base = 2`
- `duration.perQuality = 1`
- 绿色品质（1）：2 + 1×1 = 3秒
- 蓝色品质（2）：2 + 1×2 = 4秒

### Q: 如何实现"每X秒触发一次"的效果？

A: 使用定时触发器 `OnTick:X`，并在 Update 或协程中调用：
```csharp
effectModule.RaiseTick(1.0f); // 每秒调用一次
```
