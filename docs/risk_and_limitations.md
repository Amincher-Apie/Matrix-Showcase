# 当前限制与风险说明

本文说明 Matrix Showcase 中不能夸大的地方，以及公开仓库的使用边界。

## 1. 不是完整可运行工程

本仓库只保留核心自研代码和文档，不包含完整 Unity 工程所需的资源、场景、Prefab、ScriptableObject 资产和第三方插件。

不能这样表述：

- “直接 clone 就能运行完整游戏。”
- “这是完整开源版。”
- “这是完整商业项目源码。”

推荐表述：

> 这是技术展示版，适合阅读架构和核心代码，不保证直接运行。

## 2. 第三方资源不属于原创

完整项目中使用过第三方资源、商业插件、模型、贴图、音频、Shader 或字体。本仓库不重新分发这些内容，也不声明其为原创。

不能这样表述：

- “所有美术资源都是我做的。”
- “仓库里包含完整游戏资源。”
- “第三方插件也是自研的。”

推荐表述：

> 本仓库只公开自研代码和文档，第三方资源因授权原因不公开。

## 3. Unity Inspector 绑定无法从源码完全确认

Unity 项目中大量运行时关系依赖 Inspector：

- `[SerializeField]` 字段赋值。
- Prefab 上挂载的组件。
- Scene 中管理器对象的存在。
- ScriptableObject 资产之间的引用。
- Resources 路径是否和资产匹配。

因此，仅凭 Showcase 源码无法保证完整运行。

## 4. 商业插件依赖

部分代码或完整工程可能依赖商业插件或第三方插件，例如：

- Behavior Designer
- Odin / Sirenix
- Amplify Shader Editor
- DOTween
- 其他第三方资源包

Showcase 不包含这些插件。涉及插件的代码只能作为架构参考，不能单独编译运行。

## 5. 部分模块仍是原型阶段

以下内容需要谨慎表述：

- `LobbyManager`、`HeroSelector` 等部分流程可能仍带有占位逻辑。
- `QualityEffects` 中部分动作执行器属于扩展点。
- 技能表现、动画接入和远端表现可能仍有阶段性 TODO。
- 部分调试日志和历史注释需要公开前整理。
- Boss 行为树依赖完整工程和第三方插件配置。

推荐表述：

> 该项目是技术原型，核心系统已具备架构和代码实现，但部分表现、资源绑定和内容配置仍需要完整工程支持。

## 6. 网络安全边界

项目强调服务器权威，但这不等于已经完成商业级反作弊。当前可以说明：

- 关键状态变更放在服务端。
- 客户端通过 `ServerRpc` 提交操作意图。
- 服务端模块负责验证、结算和同步。

不应夸大为：

- “已经具备完整商业反作弊系统。”
- “完全无法作弊。”

## 7. 性能边界

AI 调度、对象池、PCG 失败回退等设计能体现性能意识，但 Showcase 没有提供完整性能测试报告。

可以说：

> 项目设计了 AI LOD Tick、对象池和 PCG 诊断机制，用于降低运行时开销和提升调试效率。

不建议说：

> 已经通过大规模线上压测。

## 8. 文档与编码问题

原项目中的部分文档可能存在编码显示问题或内部开发记录。发布前建议：

- 检查 `docs/project/PROJECT_OVERVIEW.md` 和 `docs/project/ARCHITECTURE.md` 的编码。
- 检查 `docs/CodingPlan/` 是否适合公开。
- 将对外说明优先指向本目录下新生成的文档。

## 9. 推荐公开说法

可用于 README 或面试：

> Matrix Showcase 是一个面向技术评审的公开版本，保留了多人 Roguelike TPS 原型中的核心自研代码和架构文档。完整 Unity 工程由于资源授权、工程体积和本地依赖原因不公开。本仓库重点展示服务器权威网络架构、确定性 PCG、任务系统、战斗数值管线、技能/Buff/品质道具系统、AI 调度和数据驱动工具链。
